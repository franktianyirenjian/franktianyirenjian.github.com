把Hexo部署到Github上

一、添加SSH
添加ssh的目的，
是为了在以后历次提交中，
不用输入密码。

1、添加SSH
1）、检测是否已有SSH

ls- - ->这里第一个字母是L小写字母。
2)、创建一个SSH key
两种创建方式：默认文件名和指定文件名
①、使用默认文件名

代码参数含义：
-t 指定密钥类型，默认是rsa，可省略。
－C设置注释文字，比如邮箱。这里c是大写。
－f指定密钥文件存储文件名。
此处省略了－f，因此，上面的命令运行后，会让你输入一个文件名，用于保存刚生成的SSH key代码，如：

你可以不输入文件名，使用默认文件名（推荐），会生成id_rsa和id_rsa.pub两个密钥文件，
查找存储这两个密钥文件快速方法是：在finder里按下cmd+shift+G，然后输入~/.ssh。
紧接着还会要求你两次输入密码，（这里的密码不是你的管理者密码，是你push文件时需要输入的密码）。
你可以不输直接按回车，那么当你push时就不用输密码，直接提交到github了，界面如下：

②、使用指定文件名

这样生成的两个文件名会是你自己指定的名字。
界面如下：

当你看到界面显示这句话时，说明SSH key已创建成功，你只需添加SSH key即可。

3)、添加SSH key到Github上去
①、找到id_rsa.pub文件，打开复制所有内容。
②、登录github账号，按以下路径找到setting：Your profile- ->Repository- - >setting，
左侧点击Deploy key点击添加，
将刚才复制粘贴到key，标
题可省略或者随意设一个，
省略则默认邮箱名，可修改数据打钩，保存，搞定。
4）、测试SSH key
①、在终端输入ssh -T git@github.com
随后会有一个警告，直接yes，回车；

②、接下来你若设置了push密码会提示你输入push密码，若没有则忽略，直接显示以下界面；

看到successful这个词儿，你就大功告成了，
若是“access denied”，表示拒绝访问，
那么你就只能在配置信息里部署非SSH访问了。
到此SSH key添加成功。
二、部署Hexo到Github
1）、配置部署信息

这里有个大坑，因为在Github中新建repository时，
系统自动填充的库名的域名后缀是.com，
当我将配置信息的后缀设为.io，
然后hexo deploy，
怎么都找不到respository。

最后才明白，直接将新建库名的域名后缀改为.io，
再次运行hexo deploy，successful！

最可气的是我再次将库名后缀改回.com，
并且配置信息仍设置为.io后缀时，
仍然可以hexo deploy成功，
而且当库名后缀为.io，配置信息后缀是.com时，
hexo deploy照样成功。
也就是排列组合后有四种方式可以hexo deploy,

respository _config.yml
.com .com
.io .io
.io .com
.com .io

百思不得姐，还请高人指点。

2)、在终端输入：hero clean
清除原来的缓存和临时文件
3）、hexo g -d或hexo d -g
生成静态文件并提交到github

一般是：https://username/username.github.io
就可以在网络搜索你的博客了，
至此博客搭建全部完毕！！
